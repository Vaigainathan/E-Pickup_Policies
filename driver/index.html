2. Information We Collect

2.1 Personal Information

Authentication Data: Phone number (account/OTP), Firebase authentication tokens, device authentication identifiers.

Profile Information: Full name, email (optional), profile photo, date of birth (if required for verification).

2.2 Location Information

Foreground: Current location during active trips; navigation routes; location when updating trip status.

Background (continuous when online/available): Real-time location updates to servers; essential for matching with nearby bookings, customer tracking, route optimization, ETAs, service coverage.

Opt-Out: Go offline in the app to stop background tracking; disable location permissions (prevents bookings).

2.3 Identity and Verification Documents

Driving License, Aadhaar Card, Vehicle RC, Insurance Certificate, vehicle photos (if required). Stored encrypted in Firebase Storage; used for verification and regulatory purposes.

2.4 Vehicle Information

Registration number, vehicle type/model/year, insurance details, vehicle photos.

2.5 Booking and Trip Data

Assigned bookings; customer contact info (name/phone); pickup/delivery addresses; trip status updates; completion records; verification photos; timestamps/durations.

2.6 Financial Information

Earnings Data: Per-trip earnings, daily/weekly/monthly earnings, commission, payment collection records, transaction history.

Payment Collection: Cash collection confirmations, timestamps, booking IDs.

Not Stored: Bank accounts (unless provided for payouts), card numbers, UPI IDs (unless provided for payouts).

2.7 Device Information

Device/Advertising ID, model/manufacturer, OS version, network info, WiFi info, app version/build, IP address, battery status (for tracking optimization).

2.8 Communication Data

Real-time chat with customers (temporary storage), push tokens, call logs (if in-app calling), support communications, booking SMS/notifications.

2.9 Images and Media

Profile photos; document photos (license, Aadhaar, RC, insurance); vehicle photos; pickup/delivery verification photos. Stored securely in Firebase Storage.

2.10 Usage and Analytics Data

If enabled: app usage, trip acceptance rates, feature interactions, session duration, error/crash reports, performance metrics, online/offline patterns. Analytics can be controlled via app settings if available.

3. How We Collect Information

Direct: Registration, profile updates, documents/photos uploaded, trip updates.

Automatic: Foreground/background location, device info, usage data, network info, battery status.

Third-Party: Firebase Authentication (phone verification), Google Maps API (navigation/geocoding), Firebase Analytics (if enabled), Socket.IO (real-time), MSG91 (OTP if used).

Communication: Booking assignments, in-app chat, earnings/payment updates, support, platform updates.

Verification & Safety: Credential verification, fraud prevention, compliance.

Legal Compliance: Responding to legal requests, regulatory requirements, platform safety.

6. Data Retention

Account Info: Retained while active; deleted within 30 days of deletion request (some info may be retained for legal/compliance).

Location Data: Background when online deleted after 90 days; active trip location retained 2 years (disputes); no historical storage beyond trip completion except trip routes.

Document Images: Retained while account active; deleted within 90 days of closure; may be retained longer if legally required.

Trip and Earnings Data: Trip/earnings/payment records retained 7 years (accounting/tax).

Chat Messages: Temporary during active trip; deleted within 30 days after completion.

Analytics: Aggregated up to 2 years; personal identifiers removed after 90 days.

7. Data Security

Encryption: TLS/SSL in transit; industry-standard at rest; document images encrypted in Firebase Storage; location data encrypted in transit.

Access Controls: Limited access; authentication required; verification team–restricted access to documents; audits.

Storage Security: Secure Firebase infra; restricted document access; encrypted backups; access logs.

Your Responsibility: Secure your device; do not share OTP/tokens; log out on shared devices; protect credentials.
